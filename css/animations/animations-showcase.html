Performance Comparison (quick)

`transform` + `opacity` usually animate on compositor layers (cheaper). `width`, `height`, `top`, `left`, and `margin` often trigger layout/reflow and extra paint (heavier).

Animation Syntax (quick)

Short syntax patterns you can mention in interviews.

Animation shorthand

name duration timing delay iteration direction fill play-state

syntax 
.box {
  animation: fade-in 600ms ease-out 0s 1 normal both running;
}

Transform in keyframes

Use `transform` + `opacity` for smooth motion.

syntax 
@keyframes enter {
  from { transform: translateY(12px); opacity: 0; }
  to { transform: translateY(0); opacity: 1; }
}

Bad syntax -> Better performance

Avoid layout-thrashing properties in animations.

bad vs good 
/* bad */
@keyframes bad-move { from { left: 0; } to { left: 120px; } }

/* better */
@keyframes good-move { from { transform: translateX(0); } to { transform: translateX(120px); } }

Small perf boost

Use `will-change` only for hot animated elements.

performance 
.moving-chip {
  will-change: transform, opacity;
}

fade-in

Fade In
animations.css

  animation: fade-in 1.6s ease-in-out infinite alternate;

text loop (right -> left)

@keyframes text-loop {
